# Design: Swaggo API Documentation for Keel HTTP Handlers

## Overview

Integrate [swaggo/swag](https://github.com/swaggo/swag) into the Keel project to generate OpenAPI 2.0 (Swagger) documentation from annotated Go handlers.

## Architecture

### Key Decisions

1. **Documentation Location**: Generated docs will live in `docs/` directory at project root (swag default)
2. **Main Entry Point**: Add general API info annotations to `cmd/keel/main.go` 
3. **Handler Annotations**: Add Swaggo comment annotations directly above each handler function in `pkg/http/`

### Swaggo Annotation Pattern

Each handler follows this pattern:

```go
// HandlerName godoc
// @Summary Brief description
// @Description Longer description
// @Tags category
// @Accept json
// @Produce json
// @Param paramName paramType dataType required "description"
// @Success 200 {object} ResponseType
// @Failure 400 {string} string "error message"
// @Router /path [method]
func (s *TriggerServer) handlerName(resp http.ResponseWriter, req *http.Request) {
```

### Handler Categories (Tags)

| Tag | Endpoints |
|-----|-----------|
| health | /healthz, /version |
| auth | /v1/auth/* |
| approvals | /v1/approvals |
| resources | /v1/resources |
| policies | /v1/policies |
| tracked | /v1/tracked |
| audit | /v1/audit |
| stats | /v1/stats |
| webhooks | /v1/webhooks/* |

### Request/Response Types

Existing types in the codebase will be reused:
- `types.Approval` - approval data
- `types.Repository` - webhook payload
- `types.AuditLog` - audit log entries
- Handler-local types: `approveRequest`, `trackRequest`, `loginRequest`, etc.

Some new response wrapper types may be needed for proper documentation.

## Makefile Integration

```makefile
update_openapi:
	@echo "++ Generating OpenAPI spec"
	go install github.com/swaggo/swag/cmd/swag@latest
	swag init -g cmd/keel/main.go -o docs
```

## Constraints

- Uses gorilla/mux router (standard net/http handlers)
- Some endpoints require authentication (noted in annotations)
- Webhooks can optionally require authentication based on `authenticatedWebhooks` flag
